# 旧系统数据导出字段对照表
> **用途**：从旧系统导出数据时，对照本表判断哪些字段是必须的，哪些可以不用导
>
> **设计逻辑**：从计算公式反向推导字段依赖，确保所有计算都能跑通
>
> **最后更新**：2026-05-12

---

## 一、逐公式字段依赖映射

| 序号 | 计算逻辑 | 必须要的原始字段 | 字段用途说明 | 优先级 |
|------|---------|-----------------|-------------|--------|
| --- | --- | --- | --- | --- |
| **基础分类层** | | | | |
| 1 | 五分类判断：<br>卡名含"热玛吉"=热玛吉<br>卡名在白名单=私教<br>卡名含"轮换"=轮换<br>其余=团课 | `card_name` 卡种名称 | **最核心字段**，所有耗课统计的基础 | 🔥 P0 |
| 2 | 是不是耗课：私教+热玛吉=是，其余=否 | `card_name` | 同上 | 🔥 P0 |
| 3 | 是不是体验课：课程名含"体验"=是 | `course_name` 课程名称 | 体验课数量统计 | 🔥 P0 |
| --- | --- | --- | --- | --- |
| **耗课统计层** | | | | |
| 4 | 总耗课节数 = COUNT(算耗课的记录) | `card_name` + `class_datetime` 上课时间 | 按月统计总耗课 | 🔥 P0 |
| 5 | 定制私教/热玛吉/轮换/团课各自数量 | `card_name` + `class_datetime` | 分类统计 | 🔥 P0 |
| 6 | 耗课金额 = 总耗课 × 190 | `card_name` + `class_datetime` | 不需要金额字段，统一定价 | 🔥 P0 |
| --- | --- | --- | --- | --- |
| **出勤会员层** | | | | |
| 7 | 出勤人数 = COUNT(DISTINCT 会员) | `member_name` 会员姓名 或 `phone` 手机号 | 去重统计 | 🔥 P0 |
| 8 | ≥2次出勤人数 = 分组后COUNT≥2 | `member_name`/`phone` + `class_datetime` | 活跃度统计 | 🔥 P0 |
| 9 | 在册会员总数 = COUNT(状态=正常) | `status` 会员状态 | 出勤率分母 | ⭐ P1 |
| 10 | 出勤率 = ≥2次出勤 ÷ 在册会员 × 100% | 以上两个字段 | 计算得出 | 🔥 P0 |
| --- | --- | --- | --- | --- |
| **效率分析层** | | | | |
| 11 | 人效 = 总耗课 ÷ 出勤人数 | 同上 | 计算得出 | - |
| 12 | 教练人效排名 = 按教练分组计算人均次数 | `coach_name` 教练姓名 + `member_name` + `class_datetime` | 教练绩效 | 🔥 P0 |
| --- | --- | --- | --- | --- |
| **销售达成层** | | | | |
| 13 | 本月销售课量 = SUM(购卡次数) | `total_sessions` 购卡次数 + `payment_date` 支付时间 | 销耗比分子 | ⭐ P1 |
| 14 | 本月续课量 = COUNT(订单类型=续费) | `order_type` 订单类型 + `payment_date` | 续费率分子 | ⭐ P1 |
| 15 | 本月销售金额 = SUM(实收金额) | `order_amount` 实收金额 + `payment_date` | 所有达成率分子 | ⭐ P1 |
| 16 | 销售目标/盈亏平衡线 | 配置字段 | 分母，新系统重录也行 | 🟢 P2 |
| 17 | 销耗比/续耗比/盈亏达成率/销售达成率 | 以上分子分母 | 计算得出 | - |
| --- | --- | --- | --- | --- |
| **RFM模型层** | | | | |
| 18 | R（最近消费天数）= 今天 - MAX(上课时间) | `member_name` + `class_datetime` | R值计算 | 🔥 P0 |
| 19 | F（近90天消费频次）= COUNT(近90天记录) | `member_name` + `class_datetime` | F值计算 | 🔥 P0 |
| 20 | M（近90天消费金额）= F × 190 | F值 | 计算得出 | - |
| 21 | RFM综合分 = R×0.4 + F×0.35 + M×0.25 | R/F/M得分 | 计算得出 | - |
| 22 | 会员分层 = 根据RFM得分判断 | RFM综合分 | 计算得出 | - |
| --- | --- | --- | --- | --- |
| **时间趋势层** | | | | |
| 23 | 每日耗课趋势 = 按日期分组COUNT | `class_datetime` | 日期提取 | 🔥 P0 |
| 24 | 星期分布 = 按周几分组COUNT | `class_datetime` | 星期提取 | 🔥 P0 |
| 25 | 时段分布 = 按小时分组COUNT | `class_datetime` | 小时提取 | 🔥 P0 |
| 26 | 同比环比 = 本月 vs 历史月份 | `class_datetime` | 月份提取 | 🔥 P0 |

## 二、旧系统导出字段总清单（按优先级排序）

### 🔥 【P0 必须导出，缺了就算不出来】

| 字段名 | 旧系统可能叫什么 | 出现在哪张表 | 影响多少个指标 |
|--------|-----------------|-------------|---------------|
| **卡种名称** | card_name / card_type / 会员卡名称 | 耗课记录表 | 影响 17 个指标 |
| **上课时间** | class_time / class_date / consume_time | 耗课记录表 | 影响 15 个指标 |
| **会员姓名** | member_name / customer_name / user_name | 耗课记录表 | 影响 10 个指标 |
| **教练姓名** | coach_name / teacher_name / coach | 耗课记录表 | 影响 2 个指标 |
| **课程名称** | course_name / class_name | 耗课记录表 | 影响 1 个指标（体验课） |
| **门店标识** | store_id / store_name / shop_id | 所有表 | 分门店统计 |

> 💡 **重要提示**：会员手机号如果有一定要导！比姓名靠谱，解决重名问题。没有的话用姓名+门店去重。

---

### ⭐ 【P1 重要字段，缺了部分算不准】

| 字段名 | 旧系统可能叫什么 | 出现在哪张表 | 影响哪些指标 |
|--------|-----------------|-------------|------------|
| **购卡次数** | total_sessions / count / times | 购卡订单表 | 销售课量 |
| **实收金额** | amount / price / pay_amount | 购卡订单表 | 销售金额、所有达成率 |
| **支付时间** | pay_time / payment_date | 购卡订单表 | 月度销售统计 |
| **支付状态** | pay_status / is_paid | 购卡订单表 | 只统计已支付的 |
| **订单类型** | order_type / type | 购卡订单表 | 续课量判断（新购/续费） |
| **会员状态** | status / is_active | 会员表 | 在册会员数、出勤率分母 |

---

### 🟢 【P2 可选字段，有更好没有也行】

| 字段名 | 为什么可选 |
|--------|----------|
| 月度销售目标 | 新系统可以重新录 |
| 盈亏平衡线 | 新系统可以重新录 |
| 教练所属门店 | 新系统可以重新配置 |
| 会员性别/年龄/生日 | 不影响计算，纯展示用 |

## 三、导出格式要求

| 要求项 | 具体说明 |
|--------|---------|
| 文件格式 | 统一导出 CSV 格式 |
| 编码格式 | UTF-8 编码（不要用GBK/ANSI） |
| 日期时间格式 | 统一用 `YYYY-MM-DD HH:MM:SS` 格式 |
| 数字字段 | 不要带千分位、单位符号（¥元次等） |
| 空值处理 | 空值直接留空，不要填「无」「NULL」「暂无」等 |
| 数据范围 | 建议导出 **近12个月** 的完整数据，用于做同比环比 |
| 特殊表 | 会员表、卡种配置表导出 **全量** |

## 六、异常情况处理

如果旧系统里某个字段确实没有，按以下优先级处理：

| 缺失字段 | 替代方案 | 影响程度 |
|---------|---------|---------|
| 会员手机号 | 用「姓名+门店」组合去重 | 🔵 基本能用，少量重名影响可忽略 |
| 订单类型 | 新系统根据「是不是该会员的第一张卡」来判断是不是续费 | 🔵 准确率90%+，可接受 |
| 会员状态 | 新系统上线后重新维护，历史出勤率可以先不补 | 🟢 影响最小，只有1个指标暂时算不了 |
| 卡种分类 | 新系统上线后统一重新配置卡种 | 🟡 一次性工作，配置完就好 |

## 🎯 最终结论

**只要导好 1 张核心表 + 2 张辅助表，所有计算就都能跑通：**

1. **耗课记录表**（最重要）：6个P0字段全导
2. **购卡订单表**：5个P1字段全导
3. **会员表**：2个P1字段全导

其他的都可以不用导！

## 七、实际数据问题重点标注
> ⚠️ **重要**：基于之前检查过的Excel文件，以下字段存在已知问题，导出时请重点检查！

| 字段名 | 已知问题 | 影响程度 | 处理建议 |
|--------|---------|---------|---------|
| **卡种名称** | 🔴 格式极不统一<br>- 同一张卡有多种写法<br>- 有的带门店前缀<br>- 有的带数字后缀<br>- 大小写/空格不一致 | 🔥 高 | 导出后必须做卡名标准化清洗，建立「别名→标准名」映射表 |
| **上课时间** | 🟡 格式不统一<br>- 有的是文本格式（如"2024/3/15"）<br>- 有的是Excel日期序列号<br>- 有的缺少时间部分（只有日期） | 🔥 高 | 导出前统一转为 `YYYY-MM-DD HH:MM:SS` 格式<br>缺少时间的默认补 `00:00:00` |
| **会员姓名** | 🟡 数据质量问题<br>- 有的带空格<br>- 有的是简称（"张姐""李哥"）<br>- 有的同名（两个"王芳"）<br>- 有的有特殊符号 | 🟡 中 | 尽量导出手机号作为唯一标识<br>没有手机号的用「姓名+门店」组合去重 |
| **教练姓名** | 🟠 部分缺失/不规范<br>- 有的记录教练名为空<br>- 有的是"教练""助教"等通用称呼<br>- 同教练不同写法（"张三""张教练"） | 🟡 中 | 教练名为空的记录不影响总耗课统计<br>只影响教练人效排名（可后续补） |
| **课程名称** | 🟢 基本可用<br>- 体验课基本都带"体验"二字<br>- 少量不规范但不影响判断 | 🟢 低 | 正常导出即可 |
| **门店标识** | 🟡 部分文件缺失<br>- 有的Excel文件名能看出是哪个门店<br>- 但表格内没有门店字段 | 🟡 中 | 导出时务必确保每条记录都有门店标识<br>可以从文件名推断后补入 |

### 数据清洗优先级清单

| 序号 | 清洗工作 | 不做的影响 | 必须做吗？ |
|------|---------|-----------|----------|
| 1 | 卡种名称标准化映射 | 五分类算错 → 总耗课算错 → 所有指标全错 | ✅ 必须做 |
| 2 | 上课时间格式统一 | 所有时间维度指标（趋势/星期/时段）全错 | ✅ 必须做 |
| 3 | 会员去重（手机号或姓名+门店） | 出勤人数不准 → 人效、出勤率全错 | ✅ 必须做 |
| 4 | 教练姓名标准化 | 教练排名不准，不影响其他指标 | ⭐ 建议做 |
| 5 | 补全门店标识 | 无法分门店统计，只能看汇总数据 | ⭐ 建议做 |

**文档说明**：本文档从计算公式反向推导字段需求，确保导出的每一个字段都是有用的，不会导出无用数据增加清洗工作量。
