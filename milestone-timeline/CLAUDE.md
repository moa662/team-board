# 个人时间轴项目 - 开发约定

## 产品定位
个人时间流的只读可视化呈现器 - **时间镜子**
> 让你看见自己的时间流向

## 硬规则（严禁修改）
1. **只读不编辑**：绝对不做任何编辑功能，用户只通过修改 JSON 来更新数据
2. **横向长轴**：保持横向布局，不改成纵向
3. **单条时间流**：里程碑和日志在同一条轴上，不用多时间流
4. **零依赖**：纯前端 HTML，不依赖任何后端服务或外部 CDN
5. **文件协议可用**：双击 index.html 直接打开，不需要 localhost

## 技术架构
```
timeline-data.json  ← 用户唯一需要编辑的文件
       ↓
build.py           ← Python 构建脚本（一键更新.bat）
       ↓
index.template.html ← HTML 模板（一般不修改）
       ↓
index.html         ← 最终产物，双击打开
```

## 颜色体系
- 🔵 #818cf8 蓝色 = 个人探索（开发、AI、知识库、设计、思考、灵感）
- 🟢 #10b981 绿色 = 公司项目（商业、运营、产品、短视频、税务、门店）
- 🟡 #fbbf24 黄色 = 待办事项（待办、代办、需求代办），同时背景高亮

## 核心功能清单
- 里程碑节点：5个锚点，按实际开发进度设置
- 日志卡片：按天分组，纵向堆叠
- 卡片折叠：点击标题展开/收起详情
- 拖拽滚动：按住时间轴区域横向拖动
- 标签筛选：顶部筛选栏，点击标签过滤
- 类型筛选：个人/公司快速切换
- 回到现在：一键滚动到最新位置
- 图例说明：底部三色标识

## 数据规范
- 所有内容写入 `timeline-data.json`
- 条目标签：用中文标签，不要用英文
- 日期格式：`YYYY-MM-DD`
- 时间格式：`HH:MM`

## 禁止事项
- 不要加编辑按钮、输入框等任何交互控件
- 不要做登录、保存、同步功能
- 不要依赖外部 API 或后端服务
- 不要引入 npm、node_modules 等前端构建工具
- 不要把 index.html 改成需要服务器才能打开

## 开发命令
```bash
# 构建
python build.py
# 或双击
一键更新.bat

# 预览
python -m http.server 8888
# 或直接双击 index.html
```
